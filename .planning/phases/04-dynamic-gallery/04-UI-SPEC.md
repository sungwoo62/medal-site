## Interaction States

### Gallery Filter (medal-site)

- **All categories visible on load** — default filter: '전체'
- **Filter click:** Instantly re-renders grid (client-side filter from fetched data, no network request)
- **Active pill:** bg-rose text-white shadow-sm
- **Inactive pill:** bg-white text-charcoal-light border border-border; hover: border-rose/40 text-rose

### Gallery Card

- **Default:** border border-border bg-white, no shadow
- **Hover:** shadow-lg transition-all duration-300; Award icon transitions from text-charcoal/8 to text-rose/25 over 500ms
- **Image loaded:** `<img>` fills aspect-[4/3] container, object-cover
- **No image (null image_url):** category gradient background + centered Award icon (size 52, text-charcoal/8)

### Loading State (while fetch is in-flight)

- Show 6 skeleton cards in the same grid layout
- Skeleton: rounded-2xl, border border-border, bg-white
- Image area skeleton: aspect-[4/3] bg-warm-gray animate-pulse
- Text skeleton: two bars at 60% and 80% width, h-3 bg-warm-gray animate-pulse, mt-3 mx-5 mb-5

### Error State

- Show a single centered error banner replacing the grid
- Style: bg-rose/10 border border-rose/20 rounded-xl p-8 text-center

Source: `src/app/gallery/page.tsx` — established hover/active patterns; loading/error are Claude's discretion per 04-CONTEXT.md

## Data Fetching Strategy

Decision: **SSR initial load + client-side filter** (Claude's discretion per D-05)

- Server Component wraps the page and fetches all gallery_items ordered by display_order ASC
- Data passed as props to a 'use client' child component that owns the filter state
- No additional network request on filter change — all items are in memory after initial load
- Rationale: gallery is read-only public data; SSR provides instant first paint and SEO; client filter is instant with no latency

Source: `04-CONTEXT.md` — D-05 marked as Claude's discretion
